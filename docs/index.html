Project Overview

Open-Source Blockchain Forensics delivers a practical pipeline for Bitcoin address clustering and transaction analysis. It focuses on transparency, reproducibility, and a clean interface that can scale to advanced analytics later.

Why it exists

Blockchain transparency is powerful, but pseudonymous addresses hide real-world entities. This project gives investigators and researchers a clear way to explore transaction patterns while keeping the process auditable and ethical.

Who it helps

  • Compliance and risk teams validating exposure.
  • Researchers studying network behavior.
  • Security teams tracing suspicious flows.

MVP Capabilities (Phase 1)

Focused, usable features that ship real value today.

Bitcoin Address Analysis

Fetch transactions, normalize data, and build clusters end-to-end.

Transparent Clustering

CIOH heuristic with evidence counts and bounded confidence scores.

Usable Outputs

JSON outputs for automation, plus optional CSV/text reports.

Core flow

Fetch → Normalize → Cluster → Score → Export

How it works

01

Collect

Retrieve transactions from Blockstream or your own Bitcoin Core node.

02

Normalize

Convert raw transaction data into a consistent internal schema.

03

Cluster

Apply CIOH to link addresses that co-spend in the same transaction.

04

Report

Export JSON for analysis and optional CSV/text reports for quick review.

Get Started

Run the MVP CLI in minutes.

python3 -m blockchain_forensics analyze-btc --address <addr> --out output.json
Default provider: Blockstream Default pagination: 5 pages
